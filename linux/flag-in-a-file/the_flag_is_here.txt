grep is a command-line utility GDSC{COMMOON_DUDE!_I'M_NOT_GOING_TO_PUT_IT_IN_THE_FIRST_LINE} for searching plain-text data sets for lines that match a regular expression. Its name comes from the ed command g/re/p (globally search for a regular      expression and print matching lines), which has the same effect.[3][4] grep was originally developed for the Unix operating system, but later available for all Unix-like systems and some others such as OS-9.[5]

<p><b><code>grep</code></b> is a <a href="/wiki/Command_line_interface" class="mw-redirect" title="Command line interface">command-line</a> utility for searching plain-text data sets for lines that match a <a href="/wiki/Regular_expression" title="Regular expression">regular expression</a>. Its name comes from the <a href="/wiki/Ed_(text_editor)" title="Ed (text editor)">ed</a> command <i>g/re/p</i> (<i>globally search for a regular expression and print matching lines</i>), which has the same effect.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup><sup id="cite_ref-etymology_4-0" class="reference"><a href="#cite_note-etymology-4">&#91;4&#93;</a></sup> <code>grep</code> was originally developed for the <a href="/wiki/Unix" title="Unix">Unix</a> operating system, but later available for all <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> systems and some others such as <a href="/wiki/OS-9" title="OS-9">OS-9</a>.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup>
</p>

<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Grep&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Before it was named, grep was a private utility written by <a href="/wiki/Ken_Thompson" title="Ken Thompson">Ken Thompson</a> to search files for certain patterns. <a href="/wiki/Doug_McIlroy" class="mw-redirect" title="Doug McIlroy">Doug McIlroy</a>, unaware of its existence, asked Thompson to write such a program. Responding that he would think about such a utility overnight, Thompson actually corrected bugs and made improvements for about an hour on his own program called <code>s</code> (short for "search"). The next day he presented the program to McIlroy, who said it was exactly what he wanted. Thompson's account may explain the belief that grep was written overnight.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup>
</p><p>Thompson wrote the first version in <a href="/wiki/PDP-11" title="PDP-11">PDP-11</a> <a href="/wiki/Assembly_language" title="Assembly language">assembly language</a> to help <a href="/wiki/Lee_E._McMahon" title="Lee E. McMahon">Lee E. McMahon</a> analyze the text of <i><a href="/wiki/The_Federalist_Papers" title="The Federalist Papers">The Federalist Papers</a></i> to determine authorship of the individual papers.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup> The <a href="/wiki/Ed_(text_editor)" title="Ed (text editor)">ed text editor</a> (also authored by Thompson) had <a href="/wiki/Regular_expression" title="Regular expression">regular expression</a> support but could not be used on such a large amount of text, so Thompson excerpted that code into a standalone tool.<sup id="cite_ref-history102_1-2" class="reference"><a href="#cite_note-history102-1">&#91;1&#93;</a></sup> He chose the name because in ed, the command <i>g/re/p</i> would print all lines matching a specified pattern.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup><sup id="cite_ref-9" class="reference"><a href="#cite_note-9">&#91;9&#93;</a></sup> <code>grep</code> was first included in <a href="/wiki/Version_4_Unix" class="mw-redirect" title="Version 4 Unix">Version 4 Unix</a>. Stating that it is "generally cited as <i>the</i> prototypical software tool", McIlroy credited <code>grep</code> with "irrevocably ingraining" Thompson's <a href="/wiki/Tools_philosophy" class="mw-redirect" title="Tools philosophy">tools philosophy</a> in Unix.<sup id="cite_ref-reader_10-0" class="reference"><a href="#cite_note-reader-10">&#91;10&#93;</a></sup>
</p>
<h2><span class="mw-headline" id="Sample_usage">Sample usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Grep&amp;action=edit&amp;section=2" title="Edit section: Sample usage">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>The following example demonstrates the output of the <code>grep</code> command given different arguments:
</p>
<div class="mw-highlight mw-highlight-lang-console mw-content-ltr" dir="ltr"><pre><span></span><span class="gp">$ </span>grep root /etc/passwd
<span class="go">root:x:0:0:root:/root:/bin/bash</span>
<span class="go">operator:x:11:0:operator:/root:/sbin/nologin</span>

<span class="gp">$ </span>grep -n root /etc/passwd
<span class="go">1:root:x:0:0:root:/root:/bin/bash</span>
<span class="go">12:operator:x:11:0:operator:/root:/sbin/nologin</span>

<span class="gp">$ </span>grep -c <span class="nb">false</span> /etc/passwd
<span class="go">7</span>
</pre></div>
<h2><span class="mw-headline" id="Implementations">Implementations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Grep&amp;action=edit&amp;section=3" title="Edit section: Implementations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>A variety of <code>grep</code> implementations are available in many operating systems and software development environments.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11">&#91;11&#93;</a></sup> Early variants included <code>egrep</code> and <code>fgrep</code>, introduced in <a href="/wiki/Version_7_Unix" title="Version 7 Unix">Version 7 Unix</a>.<sup id="cite_ref-reader_10-1" class="reference"><a href="#cite_note-reader-10">&#91;10&#93;</a></sup> The "<code>egrep</code>" variant supports an <a href="/wiki/Regular_expression#POSIX_basic_and_extended" title="Regular expression">extended regular expression</a> syntax added by <a href="/wiki/Alfred_Aho" title="Alfred Aho">Alfred Aho</a> after <a href="/wiki/Ken_Thompson" title="Ken Thompson">Ken Thompson</a>'s original regular expression implementation.<sup id="cite_ref-Huma1988_12-0" class="reference"><a href="#cite_note-Huma1988-12">&#91;12&#93;</a></sup> The "<code>fgrep</code>" variant searches for any of a list of <i>fixed</i> strings using the <a href="/wiki/Aho%E2%80%93Corasick_string_matching_algorithm" class="mw-redirect" title="Aho–Corasick string matching algorithm">Aho–Corasick string matching algorithm</a>.<sup id="cite_ref-Meurant1990_13-0" class="reference"><a href="#cite_note-Meurant1990-13">&#91;13&#93;</a></sup> Binaries of these variants exist in modern systems, usually linking to <code>grep</code> or calling grep as a shell script with the appropriate flag added, e.g. <code>exec grep -E "$@"</code>. <code>egrep</code> and <code>fgrep</code>, while commonly deployed on POSIX systems, to the point the POSIX specification mentions their widespread existence, are actually not part of POSIX.<sup id="cite_ref-14" class="reference"><a href="#cite_note-14">&#91;14&#93;</a></sup>
</p><p>Other commands contain the word "grep" to indicate they are search tools, typically ones that rely on regular expression matches. The <a href="/wiki/Pgrep" title="Pgrep"><code>pgrep</code></a> utility, for instance, displays the processes whose names match a given regular expression.<sup id="cite_ref-15" class="reference"><a href="#cite_note-15">&#91;15&#93;</a></sup>
</p><p>In the <a href="/wiki/Perl" title="Perl">Perl</a> programming language, <i>grep</i> is the name of the built-in function that finds elements in a list that satisfy a certain property.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16">&#91;16&#93;</a></sup> This <a href="/wiki/Higher-order_function" title="Higher-order function">higher-order function</a> is typically named <code><a href="/wiki/Filter_(higher-order_function)" title="Filter (higher-order function)">filter</a></code> or <code>where</code> in other languages.
</p><p>The <code>pcregrep</code> command is an implementation of <code>grep</code> that uses <a href="/wiki/Regular_expression#Perl_and_PCRE" title="Regular expression">Perl regular expression</a> syntax.<sup id="cite_ref-17" class="reference"><a href="#cite_note-17">&#91;17&#93;</a></sup> Similar functionality can be invoked in the GNU version of <code>grep</code> with the <code>-P</code> flag.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18">&#91;18&#93;</a></sup>
</p><p><a href="/wiki/Porting" title="Porting">Ports</a> of <code>grep</code> (within <a href="/wiki/Cygwin" title="Cygwin">Cygwin</a> and <a href="/wiki/GnuWin32" title="GnuWin32">GnuWin32</a>, for example) also run under <a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a>. Some versions of Windows feature the similar <code>qgrep</code> or <code><a href="/wiki/Findstr" title="Findstr">findstr</a></code> command.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19">&#91;19&#93;</a></sup>
</p><p>A <code>grep</code> command is also part of <a href="/wiki/ASCII_Corporation" title="ASCII Corporation">ASCII</a>'s <i>MSX-DOS2 Tools</i> for <a href="/wiki/MSX-DOS" title="MSX-DOS">MSX-DOS</a> version 2.<sup id="cite_ref-20" class="reference"><a href="#cite_note-20">&#91;20&#93;</a></sup>
</p><p>The <style data-mw-deduplicate="TemplateStyles:r886049734">.mw-parser-output .monospaced{font-family:monospace,monospace}</style><span class="monospaced">grep</span>, <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r886049734"/><span class="monospaced">egrep</span>, and <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r886049734"/><span class="monospaced">fgrep</span> commands have also been ported to the <a href="/wiki/IBM_i" title="IBM i">IBM i</a> operating system.<sup id="cite_ref-21" class="reference"><a href="#cite_note-21">&#91;21&#93;</a></sup>
</p><p>The software <a href="/wiki/Adobe_InDesign" title="Adobe InDesign">Adobe InDesign</a> has functions GREP (since CS3 version (2007)<sup id="cite_ref-22" class="reference"><a href="#cite_note-22">&#91;22&#93;</a></sup>), in the <i>find/change</i> dialog box<sup id="cite_ref-23" class="reference"><a href="#cite_note-23">&#91;23&#93;</a></sup> "GREP" tab, and introduced with InDesign CS4<sup id="cite_ref-24" class="reference"><a href="#cite_note-24">&#91;24&#93;</a></sup> in <i>paragraph styles</i><sup id="cite_ref-25" class="reference"><a href="#cite_note-25">&#91;25&#93;</a></sup> "GREP styles".
</p>
<h3><span class="mw-headline" id="agrep">agrep</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Grep&amp;action=edit&amp;section=4" title="Edit section: agrep">edit</a><span class="mw-editsection-bracket">]</span></span></h3>
<style data-mw-deduplicate="TemplateStyles:r1033289096">.mw-parser-output .hatnote{font-style:italic}.mw-parser-output div.hatnote{padding-left:1.6em;margin-bottom:0.5em}.mw-parser-output .hatnote i{font-style:normal}.mw-parser-output .hatnote+link+.hatnote{margin-top:-0.5em}</style><div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Agrep" title="Agrep">agrep</a></div>
<p><b><i>a</i>grep</b> (approximate grep) matches even when the text only <i>approximately</i> fits the search pattern.<sup id="cite_ref-eGrep.SunX_26-0" class="reference"><a href="#cite_note-eGrep.SunX-26">&#91;26&#93;</a></sup>
</p><p>This following invocation finds <i>netmasks</i> in file <i>myfile</i>, but also any other word that can be derived from it, given no more than two substitutions.
</p>
<pre>agrep -2 netmasks myfile
</pre>
<p>This example generates a list of matches with the closest, that is those with the fewest, substitutions listed first. The command flag <i>B</i> means <i>best</i>:
</p>
<pre>agrep -B netmasks myfile
</pre>
<h2><span class="mw-headline" id="Usage_as_a_verb">Usage as a verb</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Grep&amp;action=edit&amp;section=5" title="Edit section: Usage as a verb">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>In December 2003, the <i><a href="/wiki/Oxford_English_Dictionary" title="Oxford English Dictionary">Oxford English Dictionary</a> Online</i> added draft entries for "grep" as both a noun and a verb.<sup id="cite_ref-27" class="reference"><a href="#cite_note-27">&#91;27&#93;</a></sup>
</p><p>A common verb usage is the phrase "You can't grep dead trees"—meaning one can more easily search through digital media, using tools such as <code>grep</code>, than one could with a hard copy (i.e. one made from "dead trees", which in this context is a <a href="/wiki/Dysphemism" title="Dysphemism">dysphemism</a> for paper).<sup id="cite_ref-28" class="reference"><a href="#cite_note-28">&#91;28&#93;</a></sup>
</p>

History Before it was name/d, grep was a private utility written by Ken Thompson to search files for certain patterns. Doug McIlroy, unaware of its existence, asked Thompson to write such  a program. Responding that he would think about such a utility overnight, Thompson actually corrected bugs and made improvements for about an hour on his own program called s (short for "search"). The next day he presented the program to McIlroy, who said it was exactly what he wanted. Thompson's account may explain the belief that grep was written overnight.[6]
T     hompson wrote the >? first version in PDP-11 assembly language to help Lee E. McMahon analyze the text of The Federalist Papers to determine authorship of the individual papers.[7] The ed text editor (also authored by Thompson) had regular expression support but could not be used on such a large amount of text, so Thompson excerpted that code into a standalone tool.[1] He chose the name because in ed, the command g/re/p would print all lines matching a specified pattern.[8][9] grep was first included in Version 4 Unix. Stating that it is "generally cited as the prototypical software tool", McIlroy credited grep with "irrevocably ingraining" Thompson's tools philosophy in Unix.[10]Sample usageThe following example demonstrates the output of the grep command given different arguments:$ grep root /etc/passwdroot:x:0:0:root:/root:/bin/bashoperator:x:11:0:operator:/root:/sbin/nologin$ grep -n root /etc/passwd 1:root:x:0:0:root:/root:/bin/bash             12:operator:x:11:0:operator:/root:/sbin/nologi $ grep -c false /etc/passwd  7  Implementations       A variety of grep implementations are available in many operating systems and software development environments.[11] Early variants included egrep and fgrep, introduced in Version 7 Unix.[10] The "egrep" variant supports an extended regular expression syntax added by Alfred Aho after Ken Thompson's original regular expression implementation.[12] The "fgrep" variant searches for any of a list of fixed strings using the Aho–Corasick string matching algorithm.[13] Binaries of these variants exist in modern systems, usually linking to grep or calling grep as a shell script with the appropriate flag added, e.g. exec grep -E "$@". egrep and fgrep, while commonly deployed on POSIX systems, to the point the POSIX specification mentions their widespread existence, are actually not part of POSIX.[14]    Other commands contain the word "grep" to indicate they are search tools, typically ones that rely on regular expression matches. The pgrep utility, for instance, displays the processes whose names match a given regular expression.[15]         In the Perl programming language, grep is the name of the built-in function that finds elements in a list that satisfy a certain property.[16] This higher-order function is typically named filter or where in other languages.
 
The pcregrep command is an implementation of grep that uses Perl regular expression syntax.[17] Similar functionality can be invoked in the GNU version of grep with the -P flag.[18]


Ports of grep (within Cygwin and GnuWin32, for example) also run under Microsoft Windows. Some versions of Windows feature the similar qgrep or findstr command.[19]
A grep command is also part of ASCII's MSX-DOS2 Tools for MSX-DOS version 2.[20]

The grep, egrep, and fgrep commands have also been ported to the IBM i operating system.[21]

The software Adobe InDesign has functions GREP (since CS3 version (2007)[22]), in the find/change dialog box[23] "GREP" tab, and introduced with InDesign CS4[24] in paragraph styles[25] "GREP styles".

agrep
Main article: agrep
agrep (approximate grep) matches even when the text only approximately fits the search pattern.[26]

This following invocation finds netmasks in file myfile, but also any other word that can be derived from it, given no more than two substitutions.

agrep -2 netmasks myfile
This example generates a list of matches with the closest, that is those with the fewest, substitutions listed first. The command flag B means best:

agrep -B netmasks myfile
Usage as a verb
In December 2003, the Oxford English Dictionary Online added draft entries for "grep" as both a noun and a verb.[27]

A common verb usage is the phrase "You can't grep dead trees"—meaning one can more easily search through digital media, using tools such as grep, than one could with a hard copy (i.e. one made from "dead trees", which in this context is a dysphemism for paper).[28]
